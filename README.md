## 📖 项目介绍

你是否经历过：
* 入职第一天拉取代码报错 `Permission denied`？
* 分不清 `Merge` 和 `Rebase`，导致提交历史像“渔网”一样乱？
* 遇到代码冲突（Conflict）看着满屏的 `<<<< HEAD` 惊慌失措？
* 写错了代码不敢撤销，生怕把项目搞崩？

**本项目是一个单文件 HTML 应用**。无需安装 Node.js，无需配置服务器，双击即可运行。它通过**动态图解**、**场景模拟**和**时间轴演练**，手把手教你掌握符合一线大厂标准的 Git 工作流。

## ✨ 核心功能

* **🛡️ 安全起步**：从配置 SSH Key 到首次 Clone 的保姆级指引。
* **⚔️ 核心难点攻克**：**Merge vs Rebase** 动态可视化对比，一眼看懂区别。
* **🔥 冲突实验室**：还原真实的冲突代码块，手把手教你如何“删符号、留代码”。
* **🎭 沉浸式剧本**：独创「开发者的一天」时间轴，模拟从早晨接需求、突发插队任务到上线发布的全过程。
* **💊 后悔药大全**：涵盖 `reset`、`checkout`、`clean` 等救命回滚指令。

## 🎮 实操指南

建议按照手册左侧导航栏的顺序进行演练：

1. **入门篇**：检查你的 SSH 配置，确保你能拉取代码。
2. **进阶篇**：重点阅读 **Section 3**，理解为什么企业开发推荐使用 Rebase。
3. **高阶实战**：点击 **Section 7 (综合演练)**，跟着时间轴，在你的本地建立一个测试文件夹，真实地敲一遍命令。

> **💡 提示**：手册中的所有命令均带有“复制”按钮，且配有真实场景的终端输出模拟。
